1. Getting Started

What is Mapping Slayer?

Mapping Slayer is a professional web-based PDF annotation tool designed for creating interactive location maps. It allows users to place numbered dots on architectural drawings, floor plans, or site maps while storing detailed information about each location. The tool is particularly valuable for construction projects, facility management, signage installation, and compliance documentation.

System Requirements

Browser Requirements

  • Chrome 90+: Recommended for best performance
  • Firefox 88+: Full feature support
  • Safari 14+: Compatible with minor limitations
  • Edge 90+: Full compatibility
  • Internet Explorer: Not supported

System Requirements

  • RAM: Minimum 4GB, 8GB+ recommended for large projects
  • Storage: 1GB+ free space for downloads and temporary files
  • JavaScript: Must be enabled
  • Internet Connection: Required for initial loading only
  • File Access: Browser must allow local file uploads and downloads

First Launch Setup

  1. Access the Application: Open Mapping Slayer in your web browser
  2. Accept Disclaimer: Review and accept the software disclaimer and terms
  3. Choose Tooltips: Decide whether to enable contextual help tooltips (recommended for beginners)
  4. Upload First File: Either upload a PDF document or load an existing .mslay project file
Enable tooltips during your first session by clicking "HELP" and then "TOOL TIPS" in the control modal. They provide contextual help for every interface element. When tooltips are enabled, the button turns orange.

Supported File Formats

Format Extension Purpose Best Use
PDF Documents .pdf Source maps and plans Architectural drawings, floor plans, site maps
Mapping Slayer Projects .mslay Complete project files Saving and sharing annotated projects

PDF Requirements for Advanced Features

Live Text PDFs (Recommended)

  • Text can be selected and copied with your mouse
  • Enable Automap features for automatic dot placement
  • Support text scraping for rapid annotation
  • Usually created from CAD software or word processors

Scanned/Image PDFs (Limited Features)

  • Text appears as images and cannot be selected
  • Manual dot placement only
  • OCR scraping available but slower
  • May require external OCR processing for best results

3. Marker Types & Categories

Understanding Marker Types

Marker types (previously called sign types) are the foundation of project organization in Mapping Slayer. They define categories of locations and determine how dots appear visually. Every location dot must belong to a marker type, which controls its color, appearance, and grouping in reports and exports.

Marker Type Components

  • Code: Short identifier (2-6 characters recommended)
  • Name: Full descriptive title
  • Dot Color: Background color of location markers
  • Text Color: Color of location number text
  • Design Reference: Optional installation reference image

Creating and Managing Marker Types

  1. Add New Type: Click the [+] button in the Marker Types section
  2. Edit Code: Click the "NEW" code field and enter a short identifier
  3. Set Name: Click the name field and enter a descriptive title
  4. Choose Colors: Click the colored squares to open color pickers
  5. Upload Reference (Optional): Click the [+] square to add installation images
  6. Activate Type: Click anywhere on the marker type row to select it
Newly created marker types are now automatically selected as the active type, saving you a click in your workflow.

Reference images provide visual context for installation teams, showing exactly what equipment, signage, or fixtures should be installed at each location.

Color Selection Guidelines

Dot Color (Background)

  • High Contrast: Choose colors that stand out against your PDF background
  • Industry Standards: Red for safety, blue for information, green for emergency
  • Distinction: Ensure marker types are easily distinguishable
  • Printing: Consider how colors will appear when printed

Text Color (Location Numbers)

  • Consistency: White or black text works well with most backgrounds

Deleting a marker type removes ALL locations of that type from your entire project. This action cannot be undone through the undo system.

Editing Existing Types

  • Name Changes: Updates displays and exports immediately
  • Color Changes: All dots of that type update in real-time

The active marker type (highlighted in orange) determines what type of dot will be created when you click on the map. You can change the active type by:

  • Clicking on any marker type in the left panel
  • Using the Up/Down arrow keys (when not typing)
  • Selecting it from the Automap dropdown

4. Adding Location Dots

Manual Dot Placement

The most basic way to add locations is by clicking directly on the map. This gives you precise control over dot placement and is suitable for all types of PDFs.

  1. Select Marker Type: Choose the appropriate category from the left panel
  2. Position Cursor: Move your mouse to the desired location on the PDF
  3. Click to Place: Single left-click to create a new location dot
  4. Automatic Numbering: The dot receives the next sequential number
  5. Default Message: Initial message is "NEW LOCATION" (change via right-click)

Location Dot Properties

Core Properties

  • Internal ID: Unique system identifier (7 digits, zero-padded)
  • Location Number: User-visible identifier (4 digits, zero-padded)
  • Marker Type: Category determining color and purpose
  • X,Y Position: Exact coordinates on the page
  • Page Number: Which PDF page contains this location

Content Properties

  • Message 1: Primary description text
  • Message 2: Secondary description text
  • Notes: Detailed installation or reference information

Status Properties

  • Code Required: Boolean flag for locations requiring inspection codes
  • Vinyl Backer: Boolean flag for locations needing vinyl backing
  • Installed: Boolean flag for completion status

Collision Detection System

Mapping Slayer automatically prevents dots from being placed too close together, ensuring clean, readable layouts and preventing accidental overlap.

Collision Rules

  • Minimum Distance: (20 √ó dot size) + 1 pixel between dot centers
  • Real-time Check: Validation occurs before dot creation
  • Visual Feedback: Cursor changes or status message when collision detected
  • Size Adaptation: Collision distance adjusts with dot size slider
If you need to place dots closer together, temporarily reduce the dot size using the slider in the footer, place your dots, then resize them back up.

Automatic Numbering System

Location numbers are assigned automatically using a sophisticated system that maintains logical sequences across pages and marker types.

  1. Page-Based Sequences: Each page maintains its own numbering sequence
  2. Sequential Assignment: New dots get the next available number on that page
  3. Gap Filling: System fills gaps if locations are deleted
  4. Manual Override: Users can manually change location numbers via edit dialog
  5. Duplicate Prevention: System prevents duplicate numbers on same page

Dot Size and Scaling

The dot size slider in the footer controls the visual size of all location markers simultaneously.

Size Recommendations

  • 0.5-0.8: Dense layouts with many close locations
  • 1.0: Standard size for most applications
  • 1.2-1.5: Presentation mode for better visibility
  • 1.6-2.0: Large format printing or vision accessibility
  • 2.1-3.0: Extreme cases requiring maximum visibility

5. Automap & Text Recognition

Understanding Automap Technology

Automap is Mapping Slayer's intelligent text recognition system that automatically finds text patterns in PDFs and places location dots on matching content. This feature dramatically accelerates the annotation process for documents with repeated elements like room numbers, equipment codes, or fixture types.

Automap requires PDFs with "live text" - text that can be selected and copied with your mouse. Scanned documents or image-based PDFs will not work with Automap unless they have been processed with OCR software.

Automap Prerequisites

PDF Requirements

  • Live Text: Text must be selectable, not just images of text
  • Text Quality: Clear, properly encoded text without corruption
  • Language Support: Best results with English text
  • Font Compatibility: Standard fonts work better than decorative ones

Testing for Live Text

  1. Try to select text with your mouse cursor
  2. If you can highlight and copy text, Automap will work
  3. If text appears as solid blocks that can't be selected, use OCR scraping instead

Basic Automap Operation

  1. Select Target Marker Type: Choose the category for new dots from the dropdown
  2. Enter Search Term: Type the text you want to find in the input box
  3. Choose Search Mode: Select "Exact Phrase" or uncheck for "Contains Text"
  4. Execute Search: Click "AUTOMAP IT!" to start the process
  5. Monitor Progress: Watch the progress dialog for status updates
  6. Review Results: Check placed dots and make manual adjustments if needed

Search Modes Explained

Exact Phrase Mode (Default)

  • Behavior: Finds only exact matches of your search term
  • Case Sensitivity: Ignores case differences
  • Word Boundaries: Respects word boundaries and spacing
  • Best For: Specific codes, room numbers, equipment IDs
  • Example: Searching "101" finds "101" but not "1101" or "S101"

Contains Text Mode

  • Behavior: Finds any text containing your search term
  • Partial Matches: Matches substrings within larger text
  • Flexibility: Captures variations and compound terms
  • Best For: Equipment types, descriptive terms, partial codes
  • Example: Searching "EXIT" finds "EXIT", "EXIT SIGN", "EMERGENCY EXIT"

Advanced Automap Strategies

Start with the most specific, unique terms first. Search for "FIRE EXTINGUISHER" before searching for just "FIRE" to minimize false matches and processing time.

Progressive Refinement Strategy

  1. Test on Single Page: Navigate to a representative page before running automap
  2. Start Specific: Use complete, unique terms for initial passes
  3. Review and Adjust: Check results and refine search terms
  4. Expand Gradually: Use broader terms only after specific ones are complete
  5. Clean Up: Remove false positives and adjust placement manually

Text Clustering and Grouping

Automap uses intelligent text clustering to group related text elements into single location dots. This prevents multiple dots from being placed on multi-line descriptions or compound labels.

Clustering Parameters

  • Horizontal Tolerance: Maximum distance to group text left-to-right
  • Vertical Tolerance: Maximum distance to group text top-to-bottom
  • Automatic Optimization: System calculates best grouping distances
  • Manual Adjustment: H and V values can be fine-tuned in the scrape controls

Automap Progress and Feedback

The automap process provides detailed feedback through a progress modal that shows current status, completion percentage, and detailed activity logs.

Status Message Meaning Action Required

"Loading page content..." Extracting text from PDF Wait for completion
"Finding matches..." Searching for your term Monitor progress bar

Troubleshooting Automap Issues

No Matches Found

  • Verify Text Selection: Ensure text can be manually selected
  • Check Spelling: Automap is spelling-sensitive
  • Try Variations: Test different forms of the same term
  • Use Contains Mode: Switch from exact phrase to contains text

Too Many False Positives

  • Use Exact Phrase: Enable exact phrase matching
  • Be More Specific: Use longer, more unique search terms
  • Manual Cleanup: Delete unwanted dots after processing
  • Filter Results: Use selection tools to bulk-delete incorrect matches

Performance Issues

  • Large Documents: Process one page at a time
  • Complex Text: Use simpler search terms
  • Memory Limits: Close other browser tabs and applications
  • Browser Choice: Chrome typically provides best performance

Recent Searches and Efficiency

Mapping Slayer remembers your recent search terms for quick reuse across similar projects or when refining searches.

  1. Automatic Storage: Successful searches are saved automatically
  2. Dropdown Access: Click in the search box to see recent terms
  3. Quick Selection: Click any recent term to use it again
  4. Persistent Storage: Recent searches persist across browser sessions
  5. Smart Limits: Only the 10 most recent searches are kept

6. Text Scraping & OCR

Understanding Text Scraping

Text scraping allows you to manually select areas of the PDF and extract text content for automatic dot placement. This feature bridges the gap between manual placement and automap, giving you precise control over which text gets captured while still automating the extraction process.

H and V Tolerance Controls

The H (Horizontal) and V (Vertical) tolerance controls in the bottom-left of the interface are crucial for controlling how text is grouped during scraping operations. These values determine how the system decides whether text elements belong together or should be treated as separate locations.

Understanding Tolerance Values

  • H Tolerance (Horizontal): Controls how far apart text can be horizontally and still be grouped together. Default: 1.0 pixels
  • V Tolerance (Vertical): Controls how far apart text can be vertically and still be grouped together. Default: 25.0 pixels
  • Units: Both values are measured in pixels at the current zoom level
  • Impact: Lower values create more separate dots, higher values group more text together

Adjusting Tolerance Values

  1. Locate Controls: Find the H and V input boxes in the bottom-left corner
  2. Current Values: The boxes show the current tolerance settings
  3. Manual Entry: Click in either box and type a new value
  4. Apply Changes: Press Enter or click outside the box to apply
  5. Test Results: Perform a scrape to see how the new values affect grouping
For room numbers on floor plans, try H=50 and V=10 to group multi-line room descriptions while keeping adjacent rooms separate. For equipment lists, use H=200 and V=5 to capture full descriptions in single dots.

Basic Text Scraping (Live Text)

Use this method for PDFs with selectable text when you want more control than automap provides.

  1. Select Active Marker Type: Choose the category for new dots
  2. Position Cursor: Move mouse to one corner of the text area
  3. Begin Selection: Hold Shift + Right-click and drag
  4. Draw Selection Box: Drag to encompass the text you want to capture
  5. Release to Process: Release mouse button to start text extraction
  6. Review Results: Check placed dots and extracted text
The green selection box indicates normal text scraping mode. Make sure to encompass all related text in your selection for best grouping results based on your H and V tolerance settings.

OCR Scraping (Image Text)

For scanned documents or image-based PDFs where text cannot be selected, OCR (Optical Character Recognition) scraping uses computer vision to recognize and extract text.

  1. Select Active Marker Type: Choose appropriate category
  2. Position for OCR Area: Identify text within images or scans
  3. OCR Selection: Hold Ctrl + Shift + Right-click and drag
  4. Draw Purple Box: Purple selection box indicates OCR mode
  5. Wait for Processing: OCR analysis takes 10-30 seconds
  6. Review Recognition: Check accuracy of recognized text
OCR processing is significantly slower than regular text scraping and requires clear, high-contrast text for best results. The purple selection box indicates OCR mode is active.

Text Clustering System

Both scraping methods use intelligent text clustering to group related text elements into logical location dots. Understanding this system helps you get better results.

How Clustering Works

  • Line Detection: System first identifies horizontal lines of text
  • Horizontal Grouping: Text within H tolerance pixels is grouped left-to-right
  • Vertical Grouping: Lines within V tolerance pixels are merged into clusters
  • Spatial Analysis: Final clusters become individual location dots

Common Tolerance Settings

Document Type H Value V Value Result
Dense Floor Plans 20-50 5-15 Tight grouping, separate rooms
Equipment Lists 100-200 5-10 Full descriptions captured
Signage Schedules 50-100 20-40 Multi-line signs grouped
Scattered Labels 5-20 5-20 Individual items isolated

Optimizing Scraping Performance

Selection Techniques

  • Precise Boundaries: Draw selection boxes tightly around target text
  • Avoid Overlap: Don't include irrelevant text or graphics
  • Complete Capture: Ensure all related text is within selection
  • Consistent Size: Use similar selection sizes for similar content

Performance Optimization

  • Batch Processing: Scrape multiple related areas in sequence
  • Page-by-Page: Process one page completely before moving to next
  • Clear Cache: Refresh browser if performance degrades
  • Adequate Zoom: Zoom in enough to see text clearly but not excessively

Scraping Results and Outcomes

Result Type Condition Outcome
Single Cluster All text groups into one element One dot placed with combined text as message
Multiple Clusters Text separates into distinct groups Multiple dots placed, each with its group's text
No Text Found No selectable text in area Error message, no dots placed
Collision Detected Optimal placement conflicts with existing dot Error message, location skipped

Troubleshooting Scraping Issues

Poor OCR Results

  • Image Quality: Ensure text is clear and high-contrast
  • Selection Size: Don't make selection boxes too large
  • Text Size: Very small text may not OCR accurately
  • Font Compatibility: Simple fonts work better than decorative ones

Incorrect Grouping

  • Adjust H Value: Increase to group more horizontally, decrease to separate
  • Adjust V Value: Increase to group more vertically, decrease to separate
  • Test Settings: Try different values on a sample area first
  • Manual Splitting: Use multiple smaller scrapes for better control

12. Troubleshooting

Common Issues and Solutions

Performance Problems

Slow Response or Lag

  • Close Other Tabs: Free up browser memory
  • Reduce Dot Size: Smaller dots render faster
  • Work Page by Page: Use "Current Page Only" view mode
  • Clear Browser Cache: Remove accumulated temporary data
  • Save and Reload: Refresh the application state

Browser Crashes

  • Update Browser: Ensure you're using the latest version
  • Disable Extensions: Some extensions conflict with web apps
  • Increase Memory: Close other applications
  • Try Chrome: Generally provides best performance

Import/Export Issues

CSV Import Failures

  • Column Headers: Must match exactly - "MARKER TYPE CODE", "MESSAGE 1" or "MESSAGE", etc.
  • Encoding: Save as UTF-8 CSV to preserve special characters
  • Location Numbers: Must match existing locations exactly
  • Page Numbers: Must be valid page numbers in the PDF

PDF Export Problems

  • Large Files: Export may take several minutes - be patient
  • Memory Limits: Close other tabs for large exports
  • Download Blocked: Check browser download settings
  • Corrupted File: Try exporting smaller sections first

Text Recognition Issues

Automap Not Finding Text

  • Check PDF Type: Ensure text is selectable, not an image
  • Verify Spelling: Search terms must match exactly
  • Try Contains Mode: Less restrictive than exact phrase
  • Check Text Encoding: Some PDFs have corrupted text layers

Poor Scraping Results

  • Adjust H Value: Increase for wider text grouping
  • Adjust V Value: Increase for taller text grouping
  • Smaller Selections: Process text in smaller chunks
  • OCR Quality: Ensure good contrast for OCR mode

Display and Interface Issues

Missing Interface Elements

  • Zoom Level: Reset browser zoom to 100%
  • Window Size: Ensure adequate window dimensions
  • JavaScript: Must be enabled in browser
  • Ad Blockers: May interfere with some features

Dots Not Visible

  • Toggle Visibility: Check SHOW LOC button state
  • Dot Size: Increase size slider if too small
  • Page Navigation: Ensure you're on the correct page
  • Color Contrast: Change marker type colors if needed

Error Messages and Meanings

Error Message Cause Solution
"Missing required columns" CSV format incorrect Check column headers match exactly
"No Live Text Found" PDF is image-based Use OCR scraping mode instead
"Collision detected" Dot placement blocked Move to different location or adjust dot size
"Location not found" CSV references missing dot Check location number and page match
"Invalid file type" Wrong file format Use only .pdf or .mslay files
